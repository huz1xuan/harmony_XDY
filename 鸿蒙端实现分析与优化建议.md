# 学点云课堂鸿蒙端实现分析与优化建议

## 一、当前实现概览

### 1.1 架构设计

鸿蒙端采用 **ArkTS + WebView + XComponent** 三层架构：

```
┌─────────────────────────────────────────┐
│  WebView (H5课堂内容)                    │
│  - 加载 https://pclive.xuedianyun.com   │
│  - JS Bridge 通信 (xdyAndroid)          │
├─────────────────────────────────────────┤
│  XComponent (视频浮层)                   │
│  - 叠加在WebView之上                     │
│  - RTC视频渲染                           │
├─────────────────────────────────────────┤
│  ArkTS原生层                             │
│  - XRTC引擎管理                          │
│  - 音视频控制                            │
└─────────────────────────────────────────┘
```

### 1.2 核心功能实现

| 功能模块 | 实现方式 | 状态 |
|---------|---------|------|
| WebView容器 | ArkTS Web组件 | ✅ 完成 |

## 二、与Android端差异对比

### 2.1 WebView配置差异

| 配置项 | Android (X5WebView) | HarmonyOS (Web组件) | 差异说明 |
|-------|--------------------|---------------------|---------|
| User-Agent | 动态获取+追加 | 硬编码Android UA | 已对齐 ✅ |
| 缓存模式 | `LOAD_NO_CACHE` | `CacheMode.None` | 已对齐 ✅ |
| 媒体手势 | `setMediaPlaybackRequiresUserGesture(false)` | `.mediaPlayGestureAccess(false)` | 已对齐 ✅ |
| 缩放控制 | `setSupportZoom(false)` | `.zoomAccess(false)` | 已对齐 ✅ |
| 自适应布局 | `LayoutAlgorithm.NARROW_COLUMNS` | `.overviewModeAccess(true)` + `textZoomRatio` | 已对齐 ✅ |
| URL拦截 | `shouldOverrideUrlLoading` | `.onLoadIntercept` | 已对齐 ✅ |

### 2.2 退出课堂机制差异

**Android端实现：**
```java
// DemoClassActivity.java
public boolean shouldOverrideUrlLoading(WebView view, String url) {
    if (url.contains("xdyui2.xuedianyun.com")) {
        // 检测到管理页跳转，直接关闭Activity
        finish();
        return true;
    }
    return false;
}
```

**鸿蒙端实现：**
```typescript
// Index.ets
.onLoadIntercept((event) => {
    const url = event.data.getRequestUrl();
    if (url.indexOf('xdyui2.xuedianyun.com') >= 0) {
        // 清除登录态
        webview.WebCookieManager.clearAllCookiesSync();
        this.webviewCtrl.runJavaScript('localStorage.clear();sessionStorage.clear();');
        // 重载登录页（无法关闭Page，只能重定向）
        this.webviewCtrl.loadUrl(this.homeUrl);
        return true; // 阻断跳转
    }
    return false;
})
```

**关键差异：**
- Android可关闭Activity回到系统桌面/上一个Activity
- 鸿蒙端Page无法关闭，只能重载WebView URL
- 两者都需清除登录态防止自动跳转

### 2.3 JS Bridge消息差异

| 消息类型 | Android | HarmonyOS | 状态 |
|---------|---------|-----------|------|
| 1000 WebView加载 | `{"os":"android"}` | `{"os":"android"}` | ✅ 已对齐 |
| 1004 离开频道 | 关闭Activity | 重载登录页 | ⚠️ 行为差异 |
| 1010 设备信息 | 多字段格式 | 完全对齐 | ✅ 已对齐 |

### 2.4 视频渲染差异

| 特性 | Android | HarmonyOS |
|-----|---------|-----------|
| 渲染组件 | SurfaceView/TextureView | XComponent |
| 层级控制 | `setZOrderOnTop(true)` | `Stack` + `hitTestBehavior(Transparent)` |
| 坐标计算 | px单位 | vp单位（`px2vp`转换） |

## 三、已完成的优化

### 3.1 平台识别修复
- **问题：** 原始UA不含"Android"，H5识别为PC端
- **修复：** 硬编码Android UA字符串
- **结果：** H5正确进入学生端模式

### 3.2 设备信息对齐
- **问题：** `deviceSysName`为"HarmonyOS"，H5识别错误
- **修复：** 改为"Android"，补充多字段格式
- **结果：** 与Android端格式完全一致

### 3.3 退出课堂修复
- **问题：** 退出后跳转到管理后台
- **修复：** `onLoadIntercept`拦截+清除登录态+重载登录页
- **结果：** 回到学生登录页

### 3.4 登录态清除
- **问题：** 仅清cookie，localStorage保留导致自动登录
- **修复：** 增加`localStorage.clear()` + `sessionStorage.clear()`
- **结果：** 登录态完全清除

## 七、总结

鸿蒙端已实现与Android端**功能对等**，主要差异在于：

1. **生命周期管理**：Page无法关闭，需用WebView重定向模拟
2. **组件差异**：XComponent替代SurfaceView，Stack管理层级
3. **存储清除**：需同时处理Cookie+localStorage+sessionStorage

当前代码质量良好，建议按优先级逐步实施优化，重点关注**异常处理**和**性能优化**。
